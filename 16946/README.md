# 16946번 "벽 부수고 이동하기 4" 풀이 및 후기

| 바로가기  | <https://www.acmicpc.net/problem/16946> |
|-------|-----------------------------------------|
| 레이팅   | g2                                      |
| 푼 날짜  | 2023-08-26                              |
| 걸린 시간 | 25m                                     |

## 후기

예제 입출력이 친절하네요.

25분 중 채점에만 3분이 걸렸습니다.
테스트 케이스가 얼마나 많길래...

## 풀이

문제에서 구하고자 하는 값은, 각 노드별 (인접한 영역의 넓이 총합 + 1) % 10 입니다.
인접한 영역의 넓이는 노드의 상하좌우 노드가 속한 영역의 넓이입니다.

일단 모든 노드에 대해, 속한 영역의 넓이를 모두 구해야 합니다.
아무 탐색이나 쓰면 됩니다.
저는 일단 각 노드의 영역 번호를 기록하고, 영역 번호에 따른 넓이를 기록했습니다.
영역 번호는 이터레이션에 대해 유니크한 값으로 통일하면 됩니다.
넓이의 경우, 영역에 속하는 노드를 새로 찾았을 때 ++합니다.
이렇게 하면 그 영역에 속한 임의의 좌표가 주어졌을 때 넓이를 바로 얻을 수 있습니다.

이제 답을 구해봅시다.
물론 빈칸은 그냥 0을 바로 출력하면 됩니다.
이때 인접한 2개 이상의 영역이 서로 같은 것일 수도 있으므로, 인접한 영역 번호들에 distinct (toSet) 적용을 해야 합니다.

최종 시간 복잡도는 O(N * M) 이지만 노드 당 인접 노드 방문하는 연산이 많아서 약간 오래 걸립니다.
StringBuilder로 출력 최적화만 했는데 가까스로 통과했습니다.
